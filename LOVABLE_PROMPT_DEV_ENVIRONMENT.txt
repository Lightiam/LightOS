## BACKEND FUNCTIONALITY TO SIMULATE

### Build Process Simulation
When user clicks "Build App":

1. **Phase 1: Understanding (5 seconds)**
   - Show animated thinking indicator
   - Display: "Analyzing prompt..."
   - Log: "✓ Understood: Building a [type] application"

2. **Phase 2: Planning (10 seconds)**
   - Display: "Creating app plan..."
   - Generate and show JSON plan in right panel
   - Log each item:
     * "✓ Identified 5 pages"
     * "✓ Designed 3 data models"
     * "✓ Planned 8 API endpoints"

3. **Phase 3: Building (30 seconds)**
   - Progress bar from 0-100%
   - Simulate file creation:
     * "Creating package.json..."
     * "Generating React components..."
     * "Setting up FastAPI backend..."
     * "Configuring database models..."
   - File counter incrementing: "12/24 files created"

4. **Phase 4: Installing (15 seconds)**
   - Display: "Installing dependencies..."
   - Mock npm install output
   - Progress bar

5. **Phase 5: Building (10 seconds)**
   - Display: "Compiling app..."
   - Mock Vite build output
   - Show "Build complete!" with success animation

6. **Phase 6: Starting (5 seconds)**
   - Display: "Starting dev server..."
   - Show URLs:
     * "Frontend: http://localhost:5173"
     * "API: http://localhost:8000"
   - Success animation
   - "Open Preview" button appears

## USER FLOWS

### Flow 1: First Time User
1. Lands on home page
2. Sees hero section with quick start
3. Types app idea in text area
4. Clicks "Build App"
5. Redirects to /build with progress
6. Watches real-time build progress (2-3 minutes simulation)
7. Build completes with success animation
8. Clicks "Open Preview"
9. Sees working app in split-screen preview
10. Can edit code and see hot reload

### Flow 2: Template User
1. Navigates to /templates
2. Browses available templates
3. Clicks "Use Template" on React+FastAPI
4. Modal opens with customization
5. Enters app description
6. Clicks "Build from Template"
7. Redirects to /build
8. Build starts with template as base
9. Preview shows customized template

### Flow 3: Returning User
1. Lands on home page
2. Sees "Recent Projects" section
3. Clicks on existing project
4. Opens in preview page
5. Can continue editing
6. Can restart build with new prompt
